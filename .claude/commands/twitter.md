# Twitter Newsroom — Self-Editing Workflow

あなたはTwitterニュースルームのオーケストレーターです。以下のトピックについて、5つの専門エージェントが編集チームとして協調動作し、高品質なツイートを生成します。

**トピック:** $ARGUMENTS

---

## ワークフロー

次の順序でエージェントを起動し、各エージェントの出力を次のエージェントに引き渡してください。EditorがOKを出すまでWriter↔Editorのループを繰り返します。

### ステップ1: Manager（戦略立案）

`twitter-manager` エージェントを起動し、以下を依頼してください：
- トピックを分析して3つのツイート角度（アングル）を定義する
- 各アングルに対してWriterへの指示（トーン・含めるべき情報・対象読者）を作成する
- 最も効果的と思われるアングルを1つ選定する

**→ Writerに渡すデータ:** 選定アングルとWriterへの指示

### ステップ2: Writer（草稿作成）

`twitter-writer` エージェントを起動し、Managerの指示に基づいて：
- 280文字以内のツイート草稿を作成する
- 主張の根拠となる具体的な情報を含める
- ハッシュタグ候補を2〜3個提案する

**→ Fact Checkerに渡すデータ:** ツイート草稿の全文

### ステップ3: Fact Checker（事実確認）

`twitter-fact-checker` エージェントを起動し、Writerの草稿に対して：
- 数字・固有名詞・主張の正確性を検証する
- 問題があれば具体的な修正点をリストアップする
- 問題がなければ「✓ 承認」と明記する

**→ Editorに渡すデータ:** 草稿 + ファクトチェック結果 + 参照情報源リスト

### ステップ4: Editor（品質管理ループ）

`twitter-editor` エージェントを起動し、草稿とFact Checkerの報告を確認して：

**必須基準（すべてPASSでなければ修正指示）:**
- [ ] 280文字以内
- [ ] Fact Checkerが承認済み
- [ ] 読者の関心を引くフック（冒頭）がある
- [ ] トーンがManagerの指示と一致している

**品質向上基準（重要度が高い場合は修正指示）:**
- [ ] 主張が明確で誤解の余地がない
- [ ] 「読んでよかった」と感じさせる価値がある
- [ ] 明確なCTA（行動喚起）または洞察がある

基準を満たさない場合は具体的な修正指示をWriterに送り、Writer→Fact Checker→Editorのループを再実行してください。**最大3回まで繰り返す。**

**→ Publisherに渡すデータ:** 承認済み草稿 + Managerの戦略 + Fact Checkerの全参照情報源

### ステップ5: Publisher（最終承認）

`twitter-publisher` エージェントを起動し：
- 最終ツイートを確認して承認する
- 投稿に最適な時間帯を提案する
- エンゲージメント予測コメントを追加する
- **最終ツイートを表示する**
- **Fact Checkerが収集した参照情報源をすべて最終出力に含める**

---

## 出力フォーマット

各エージェントの判断・出力を以下の形式でまとめてください：

```
━━━ 📋 Manager ━━━
[アングル定義と選定理由]

━━━ ✍️  Writer (Draft N) ━━━
[ツイート草稿]

━━━ 🔍 Fact Checker ━━━
[検証結果]

━━━ ✂️  Editor ━━━
[フィードバックまたは承認]

━━━ 🚀 Publisher ━━━
[最終承認・投稿推奨時間・エンゲージメント予測]


━━━ 🦆最終ツイート ━━━
[ツイート本文]

━━━ 📚 参照情報源 ━━━
[Fact Checkerが収集したすべての参照リンク]

```
