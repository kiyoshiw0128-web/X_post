# Twitter Newsroom — Self-Editing Workflow

あなたはTwitterニュースルームのオーケストレーターです。専門エージェントが編集チームとして協調動作し、高品質なツイートを生成します。

**入力:** $ARGUMENTS

---

## ワークフロー

### ステップ0: モード選択

まず、ユーザーに投稿モードを選択してもらう。

**選択肢を提示:**
```
どちらのモードで投稿しますか？

1️⃣ ニュースモード — トレンド・最新ニュースを起点に投稿を作成
2️⃣ 気づきモード — あなたのVaultナレッジから「気づき」を抽出して投稿を作成

番号で選んでください:
```

ユーザーの選択に応じて以下の分岐を行う：

---

#### モード1: ニュースモード

`twitter-strategist` エージェントを起動し、**トレンド調査**ベースでテーマを選定する（従来通り）。
トピックが `$ARGUMENTS` で指定されている場合はそのトピックを使用する。

→ ステップ1 へ進む

---

#### モード2: 気づきモード

`twitter-strategist` エージェントを **気づき抽出モード** で起動する。以下の手順を実行：

1. **Vaultスキャン**: 以下の3つのフォルダから直近のファイルを読み込む
   - `02_Knowledge/articles/` — ナレッジ記事
   - `03_Projects/` — プロジェクト関連ドキュメント
   - `04_Creative/` — note記事・壁打ちログ・小説

2. **気づき候補リスト作成**: 読み込んだ内容から、以下の観点で「気づき」を3〜5個抽出する
   - 自分の実体験から得た学び
   - 既存の常識に対する新しい視点
   - 試行錯誤の中で発見したパターン
   - 異なる分野の知識を組み合わせた洞察

3. **ユーザーに選択を提示**: 抽出した気づき候補をリスト表示し、どれを投稿するか選んでもらう

4. **テーマ決定**: 選ばれた気づきをテーマとし、関連するVaultのコンテキスト（元記事の要約）を添えて Manager に引き渡す

→ ステップ1 へ進む

---

### ステップ1: Manager（戦略立案）

`twitter-manager` エージェントを起動し、以下を依頼してください：
- テーマを分析して3つのツイート角度（アングル）を定義する
- 各アングルに対してWriterへの指示（トーン・含めるべき情報・対象読者）を作成する
- 最も効果的と思われるアングルを1つ選定する
- **気づきモードの場合:** Strategist から渡されたVaultコンテキスト（元記事の要約）を活用し、本人の実体験・知見に根ざしたアングルを優先する

### ステップ2: Writer（草稿作成）

`twitter-writer` エージェントを起動し、Managerの指示に基づいて：
- 280文字以内のツイート草稿を作成する
- 主張の根拠となる具体的な情報を含める
- ハッシュタグ候補を2〜3個提案する

### ステップ3: Fact Checker（事実確認）

`twitter-fact-checker` エージェントを起動し、Writerの草稿に対して：
- 数字・固有名詞・主張の正確性を検証する
- 問題があれば具体的な修正点をリストアップする
- 問題がなければ「✓ 承認」と明記する

### ステップ4: Editor（品質管理ループ）

`twitter-editor` エージェントを起動し、草稿とFact Checkerの報告を確認して：

**品質基準（すべて満たすまでループ）:**
- [ ] 280文字以内
- [ ] Fact Checkerが承認済み
- [ ] 読者の関心を引くフック（冒頭）がある
- [ ] 明確なCTA（行動喚起）または洞察がある
- [ ] トーンがManagerの指示と一致している

基準を満たさない場合は具体的な修正指示をWriterに送り、Writer→Fact Checker→Editorのループを再実行してください。**最大3回まで繰り返す。**

### ステップ5: Publisher（最終承認）

`twitter-publisher` エージェントを起動し：
- 最終ツイートを確認して承認する
- 投稿に最適な時間帯を提案する
- エンゲージメント予測コメントを追加する
- **最終ツイートを表示する**
- **Fact Checkerが収集した参照情報源をすべて最終出力に含める**
- **ユーザーに最終確認を取る（投稿前に必ず承認を得ること）**

### ステップ5.5: Googleカレンダー登録

`mcp__8d77c23d-ae2c-4ff9-a00e-f74ee1a366f4__gcal_create_event` を使い、Publisherが推奨した投稿時間をGoogleカレンダーに登録する：

- **イベントタイトル**: `📝 X投稿: {トピックを短く}`
- **開始時間**: Publisherが推奨した最推奨投稿時間（JST）
- **終了時間**: 開始時間の30分後
- **説明（description）**:
  ```
  【投稿本文】
  {最終ツイート本文（ハッシュタグ込み）}

  【参照情報源】
  {Fact Checkerが収集したすべての参照URL}
  ```
- **リマインダー**: 投稿30分前にpopup通知を設定
- **タイムゾーン**: `Asia/Tokyo`

登録完了後、カレンダーイベントのURLをユーザーに表示する。

---

### ステップ5.6: Threads投稿

ユーザーの承認後、`mcp__threads-mcp` を使ってThreadsに投稿する：

1. **本文投稿**: `mcp__threads-mcp__publish_thread` で最終ツイート本文（ハッシュタグ込み）を投稿する
   - Threadsは **500文字制限** のため、本文が収まることを確認する
   - 収まらない場合は本文を分割して投稿する
2. **参照情報源リプライ**: `mcp__threads-mcp__create_reply` で、投稿した本文の `id` に対してリプライを作成する
   - Fact Checkerが収集した参照情報源URLをリプライとして追加する
   - 500文字を超える場合は複数リプライに分割する
3. **投稿直後はリプライが失敗することがある** ため、失敗時は数秒待ってリトライする

### ステップ6: Obsidian保管

最終承認済みツイートを Obsidian Vault に保存する：
- 保存先: `/Users/watanabekiyoshi/Documents/MyVault/02_Knowledge/X_posts/`
- ファイル名: `{今日の日付}_{トピックを短く英語で}.md`
- 保存内容:
  - ツイート本文
  - ハッシュタグ
  - 投稿推奨時間帯
  - 参照情報源リスト
  - Manager が定義したアングル

### ステップ7: Slack通知

`mcp__claude_ai_Slack__slack_send_message` を使い、承認済みツイートをSlackに投稿する：
- チャンネルが指定されていない場合は `mcp__claude_ai_Slack__slack_search_channels` で「general」または「雑談」チャンネルを探して投稿する
- メッセージ形式:
  ```
  📝 *X投稿案* が完成しました！

  {最終ツイート本文}

  ⏰ 推奨投稿時間: {推奨時間帯}

  📚 参照情報源:
  {Fact Checkerが収集したすべての参照リンク（1行1件）}
  ```
